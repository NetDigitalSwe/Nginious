Chapter 10. Multipart content

10.1. Overview

Nginious has built in support for parsing and handling multipart content. If a Content-Type header with value multipart is found in a request the server automatically extracts files and parameters and makes them available to HTTP services through the HttpRequest instances.

Uploaded files are available through the getFiles() and getFile(String name) methods of the HttpRequest class.

Parameters are available through the standard parameter handling methods of the HttpRequest class. These methods are the getParameter(String name), getParameterNames() and getParameterValues(String name) methods.

10.2. Progress tracking

Nginious provides classes and methods for tracking the progress of a multipart upload containing a large file. The following outlines the steps necessary to track the progress of a file upload.

Name Description
Initialization Before starting a file upload the prepareUploadTracker() method must be called. This method sets a tracking cookie which makes it possible for the server to associate a file upload with requests for the upload progress. This must be done before the file upload starts, for example in the HTTP service that serves the form for starting the file upload.
Upload A HTTP service must be implemented for accepting and processing the uploaded file. This can be done by subclassing the HttpService class and method executePost(HttpRequest req, HttpResponse resp).
Tracker Implement a HTTP or REST service which returns the status of an ongoing file upload. The service must call the getUploadTracker() method to retrieve the current status of any ongoing file uploads. This information can then be serialized into json format for sending to the client.
