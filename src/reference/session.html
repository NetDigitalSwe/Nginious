Chapter 9. Session management

9.1. Overview

HTTP is a stateless protocol where any state information disappears when a response has been sent for a request. Nginious provides sessions as a way to store information across several requests. Session can be stored in memory on the server or serialized as cookies. See section 4.2 Starting and stopping for how to configure Nginious for session management.

9.2. In memory sessions

With in memory sessions the server stores sessions in a memory map. A unique key is generated for each created session. The key is set as a cookie which is sent to the client. The client sends the cookie in later requests which the server uses to lookup the clients session from the map.

9.3. Cookie sessions

With cookie sessions the server serializes, encrypts and base64 encodes sessions and stored them in one or more cookies sent to the client. When the client sends a request to the server the cookies are part of the request and can be deserialized by the server to retrieve state information from the session.

Cookie sessions require more processing from the server since sessions have to serialized in responses and deserialized on requests. On the other hand session information is not stored in server memory and therefore survive a server restart. Sessions also become available on all servers in a multi server setup without the need to persist sessions in a shared database or similar.
