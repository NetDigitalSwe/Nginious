Chapter 2. Overview

2.1. HTTP server

Nginious implements a full featured HTTP server on top of an asynchronous I/O engine. It supports HTTP/0.9, HTTP/1.0 and HTTP/1.1 (all features except multi ranges). The asynchronous I/O engine enables the HTTP server to scale to many simultaneous connections and also enables the server to support long lived connections.

2.2. Web applications

Developers write controllers to implement web based applications in Nginious. These are packaged together with static content into web applications in a similar way as in servlet based HTTP containers. A controller is a Java class which is annotated with the Controller and Request annotations.

2.3. Web sockets

Support for web sockets. Developers write web socket controllers to implement behavior. These are packaged together with static content into web applications. See the Controller and Message annotations.

2.4. RESTful controllers

Nginious provides support for REST based controller. JSON or XML content sent to REST controllers is automatically deserialized into beans and result beans from controllers is automatically serialized into JSON or XML content. Serialization and deserialization is done based on the values of Content-Type and Accept headers.

2.5. Services

Services publish an API or run background services in a separate thread in a web application. Controllers can access services by declaring them in its methods. See the Service annotation for information about how to declare a Java class as a service.

2.6. XSP

XSP is a template language for generating HTML pages from dynamic content. Pages are translated directly into classes using bytecode translation. The javac compiler is not used. This makes the translation very fast and makes it possible for developers to make changes and reload pages in their browser to view changes almost instantly.

2.7. Multipart content

The HTTP server has built in support for handling multipart content. It also exposes an API to developers making it easy to develop web applications with support for file uploads. The API makes it possible to develop functionality for tracking the progress of file uploads.

2.8. Eclipse plugin

Nginious is integrated with Eclipse through a plugin which makes it possible to run the Nginious server inside Eclipse. The plugin also provides special project layouts and wizards for creating Nginious projects.

2.9. Server administration

Nginious provides REST based services for deploying web applications to a remote server. This is done as a multipart upload to a specific admin URL. Nginious servers keep up to 10 versions of a deployed web application. A web application can be rolled back to the previous version if problems or errors are discovered with the latest deployed version.

2.10. Resource reloading

At the heart of Nginious is a flexible resource reloading mechanism which consists of a dynamic class loader and several other mechanisms to reload XSP templates and controllers. If a class, jar file or XSP template is modified it is automatically reloaded without the need to restart Nginious or redeploy web applications. This together with the Eclipse plugin provides a powerful development environment with almost no wait time between code changes and updates in Nginious.

2.11. Ant integration

Two ant tasks are provided. One for compiling XSP templates into Java classes and one for deploying web applications to a remote Nginious server.

2.12. Session management

Two HTTP session management strategies are provided. In-memory sessions which uses a tracking cookie to identify sessions. The other option is cookie based session which serializes the sessions contents into a cookie.

2.13. Statistics

Nginious tracks basic HTTP request and web socket session statistics per minute. Statistics information can be accessed over HTTP from Nginious servers.
